# 海賊の宝石分配問題：マルチエージェント強化学習

このプロジェクトは、ゲーム理論における古典的な「海賊の宝石分配問題」に、不確実性（確率的な提案権の移行）と有限のリスク（命の重さ）を導入したマルチエージェント強化学習（MARL）シミュレーション環境です。
5人のAIエージェントが、自律的に「政治的駆け引き」や「強者の傲慢・弱者の妥協」といった戦略を学習・創発する過程を検証することを目的としています。

---

## 🎯 環境ルール

- **エージェント数**: 5人（A, B, C, D, E）
- **宝石の総数**: デフォルト 10個（高速化・検証用として 3〜5個への変更も可能）
- **報酬とペナルティ**:
  - **提案が可決された場合**: 各人が獲得した宝石の数がそのまま報酬となります。
  - **提案が否決された場合**: 提案者は海に落とされ死亡し、ペナルティ（$-L$）を受けます。
- **不確実性（権力ウェイト）**:
  - 提案者が死亡した場合、次の提案者は残りの生存者の中から「権力ウェイト」に比例した確率、または固定の順番で選出されます。

---

## 🧠 エージェントの最適化アーキテクチャ

本プロジェクトでは、最新の深層強化学習アルゴリズムを用いてAIエージェントを最適化しています。

### 1. ゲームのモデル化 (ニューラルネットワーク)
エージェントの意思決定（脳）には、**行動マスク（Action Masking）機能付きの Double DQN（Deep Q-Network）** を採用しています。
- **入力（状態）**: 「生存フラグ」「権力ウェイト」「現在の提案者（One-hot）」「現在の分配案」を結合した $4 \times N$ 次元のベクトルを入力とします。
- **ネットワーク構造**: 隠れ層 `[128, 128]` の多層パーセプトロン（MLP）を通し、各行動のQ値（価値）を出力します。
- **Action Masking**: 「すでに死亡しているエージェントに配る」「フェーズ外の行動をとる」といったルール違反の行動のQ値を強制的に $-10^9$ などの極小値に書き換え、有効な行動のみを探索させます。

### 2. 学習と評価のサイクル
5人の独立したエージェントは、以下のサイクルで統括・学習されます。
- **MultiAgentPolicyManager**: 手番に応じて5つの独立したDQNポリシーを自動で切り替えます。
- **並列学習 (SubprocVectorEnv)**: 複数のゲーム環境をマルチプロセスで同時に走らせ、経験データを高速にリプレイバッファへ蓄積し、ネットワークの重みを更新します。
- **高速評価 (Inference Mode)**: 学習後の最終テストでは、$\epsilon$-greedyのランダム要素を排除し、PyTorchの `inference_mode` を用いて最速でゲームをシミュレーションします。

## 📦 モジュール構成

### 主要な外部ライブラリ
- **PettingZoo (`pettingzoo`)**: マルチエージェント環境（AECEnv）の標準インターフェース。
- **Tianshou (`tianshou`)**: 高機能な強化学習プラットフォーム。DQNアルゴリズム、並列環境、ポリシーマネージャーを提供。
- **PyTorch (`torch`)**: ニューラルネットワークの構築と最適化計算を担うディープラーニングフレームワーク。

### 内部モジュール (ファイル構成)
- `env.py`: PettingZooを継承し、海賊ゲームのルールやAction Maskを定義した環境モデル。
- `network.py`: PyTorchを用いたDQNのネットワークアーキテクチャ（MLP）。
- `train.py`: Tianshouを用いて並列環境でデータを集め、モデルを訓練する学習ロジック。
- `eval.py`: 学習済みモデルを読み込み、推論モードで最終ゲームを実行する評価ロジック。
- `run_experiment.py`: 設定の読み込みから学習・評価・ログ出力までを一元管理するメインスクリプト。
- `export_onnx.py`: 学習済みのPyTorchモデルを汎用的なONNX形式にエクスポートするユーティリティ。

## 🛠️ 環境構築手順

1. Python 3.8以上の環境を用意します。
2. 仮想環境（`venv`）を作成し、アクティベートします。

**Windows の場合:**
```bash
python -m venv .venv
.venv\Scripts\activate

```

**macOS / Linux の場合:**

```bash
python3 -m venv .venv
source .venv/bin/activate

```

3. 以下のコマンドで必要なパッケージをインストールします。

```bash
pip install -r requirements.txt

```

## 🚀 実行方法

### 1. 環境設定 (`config.json`)

実験を始める前に、プロジェクトルートにある `config.json` を編集してパラメータを調整します。

```json
{
    "num_agents": 5,
    "total_gems": 10,
    "L": 1.5,
    "agent_weights": [20.0, 10.0, 5.0, 2.0, 1.0],
    "train_epochs": 50,
    "fixed_order": true
}

```


### 2. 実験の実行

設定が完了したら、以下のコマンドを実行します。学習から評価ゲームの実行、結果の保存までが一括で行われます。

```bash
python run_experiment.py

```

### 3. 出力ファイルの確認

実験を実行すると、自動的に以下のフォルダに連番（`_n`）付きで結果が生成・保存されます。

* **`result/result_n.txt`**: 環境設定や最終的な報酬結果などをまとめたサマリーレポート。
* **`models/policy_n.pth`**: 学習済みネットワークの重み（統合モデルファイル）。
* **`log/log_learning_n.txt`**: Tianshouによるエポックごとの学習ログ。
* **`log/log_eval_n.txt`**: 学習後のエージェントによる、分配案や投票行動の詳しい駆け引きログ。

### 4. (オプション) ONNXモデルへのエクスポート

他のシステムや軽量な推論環境でAIを動かしたい場合は、以下のスクリプトを実行してモデルをONNX形式に変換できます。

```bash
python export_onnx.py

```

実行後、`models/onnx/` ディレクトリに各エージェントの `.onnx` ファイルが生成されます。
